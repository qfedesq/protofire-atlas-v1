# Chain Page Layout

Protofire Atlas chain pages should read like a strategy memo, not a dashboard.

## Required reading order

1. Chain header
2. Primary score
3. Score composition
4. Improvement path
5. Competitive context
6. Global context
7. CTA

## Layout rules

- keep the page flat and document-like
- avoid explanatory multi-column layouts
- use vertical analytical entries for blockers and activations
- use tables for score composition and other dense comparative content
- prefer borders, labels, and spacing over cards or chips

## Section toggles

Major titled sections use [`components/ui/expandable-section.tsx`](/Users/qfedesq/Desktop/Atlas/components/ui/expandable-section.tsx).

Rules:

- section title is always visible
- body can be expanded or collapsed
- keep disclosure subtle and analytical
- default open only where the section is essential to first-glance understanding

## Economy label normalization

Chain pages must show one normalized economy/module summary line.

Examples:

- `AI Agents Readiness: Registry, Payments, Indexing, Security`
- `DeFi Infrastructure Readiness: Liquid Staking Infrastructure, Lending Infrastructure, Liquidity Layer, Oracle Infrastructure, Indexing Layer`

This formatting is generated by [`lib/utils/economy-summary.ts`](/Users/qfedesq/Desktop/Atlas/lib/utils/economy-summary.ts).
